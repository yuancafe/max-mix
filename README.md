# max-mix

A practical MiniMax multimodal skill pack for IDE/CLI agents.

`max-mix` packages image, video, music, TTS, voice-clone, search, vision, and text-chat workflows into reusable skills.

## What’s Included

- `max-image-gen` - image generation (`image-01`)
- `max-video-gen` - video generation (Hailuo series)
- `max-tts` - text-to-speech (`speech-hd`)
- `max-voice-clone` - upload + clone voice workflow
- `max-music-gen` - music generation (`music-2.5/2.6`)
- `max-music-cover` - cover generation (`music-cover`)
- `max-search` - web search (`coding-plan-search`, direct-first)
- `max-vision` - image understanding/OCR (`coding-plan-vlm`, direct-first)
- `max-text-chat` - text/coding chat (`MiniMax-M*`)

## Install

```bash
bash ./install.sh
bash ./check-deps.sh
```

## Usage Pattern

Most generation skills support a two-step flow:

1. `plan` – collect parameters and save a reusable JSON plan
2. `run` – execute from the JSON plan (optionally override fields)

Example:

```bash
python3 skills/max-video-gen/scripts/max_video_gen.py plan --interactive
python3 skills/max-video-gen/scripts/max_video_gen.py run --plan-file <plan.json>
```

For fast tasks, `max-search` and `max-vision` are direct-first:

```bash
python3 skills/max-search/scripts/max_search.py --q "MiniMax latest updates"
python3 skills/max-vision/scripts/max_vision.py --image ./shot.png --prompt "Extract text"
```

## Configuration

Before using the skills, authenticate MiniMax once:

```bash
mmx auth login --api-key <YOUR_MINIMAX_API_KEY>
mmx quota show
```

The scripts read credentials in this order:

1. `MINIMAX_API_KEY` environment variable
2. `~/.mmx/config.json` (generated by `mmx auth login`)

## Notes for Contributors

- Do not commit personal credential files.
- Keep examples key-free and environment-agnostic.
- When adding new skills, follow the same `plan/run` or `direct-first` conventions.
